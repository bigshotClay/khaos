background: |
  Vulkan operations return VkResult, a C integer code that callers in higher-level languages can silently ignore. VulkanOutcome is a sealed class hierarchy that models the meaningful terminal states of a Vulkan command — enabling compile-time exhaustive handling at every call site. Terminal states are derived from domain semantics rather than raw VkResult codes: successful completion with a produced resource; degraded success (e.g., suboptimal swapchain); recoverable failure (e.g., out-of-pool memory, timeout); and unrecoverable failure (e.g., device lost, surface destroyed). The primary consumer is the renderer — call sites in the main render loop dispatch on outcome variant to determine whether to proceed, retry, or escalate.

# Story Context

## PR #39 Review Findings

Two issues raised by Copilot review on 2026-04-24:

### Issue 1 — Negative compile tests assert exit code only (VulkanOutcomeSpec.kt:41)

The tests that verify non-exhaustive `when` expressions fail to compile only assert `exitCode == COMPILATION_ERROR`. This passes even if the compilation fails for an unrelated reason (e.g., missing import, classpath error). The test should also assert the diagnostics contain the expected sealed-exhaustiveness message so the test specifically verifies the enforcement mechanism.

**Resolution in v3:** AC2 requires that the compiler diagnostic output be inspected for the sealed-exhaustiveness error message. Implementation should check `compilation.messages` (or equivalent kctfork API) for the relevant error text.

### Issue 2 — Smart-cast vs unchecked cast star-projects generic (VulkanOutcomeSpec.kt:46–55)

The test casts `outcome as VulkanOutcome.Success` without a type argument. This star-projects the generic, making `resource` resolve to `Any?` at compile time rather than the declared `T`. The acceptance criteria require typed access after narrowing — the test should use a smart-cast (`if (outcome is VulkanOutcome.Success)`) or an explicit typed cast (`as VulkanOutcome.Success<String>`) so `resource` is statically `T`.

**Resolution in v3:** AC3 explicitly requires smart-cast or typed narrowing. Implementation should replace unchecked casts with `is`-checked smart-casts and declare the result variable with the explicit type (e.g., `val resource: String = outcome.resource`) to prove static typing is preserved.
