## New Findings

### [HIGH] env.clear() regression: PATH not restored — git binary unreachable on CI
**Reviewers:** The Infiltrator (INF-L2-01), The Coroner (COR-L2-01)  
**Location:** `khaos-test-harness/src/test/kotlin/khaos/spike/SpikeDecisionDocument.kt:24`  
**Category:** security/operability  

`pb.environment().clear()` was correctly added to isolate the git subprocess. However, `PATH` is not restored afterward. Without `PATH`, `ProcessBuilder("git", ...)` cannot locate the git binary on non-standard runners (Homebrew macOS: `/opt/homebrew/bin/git`; Alpine: `/usr/bin/git`). `ProcessBuilder.start()` throws `IOException` uncaught, crashing the test rather than returning `false`.

The spike specs do this correctly: `SpikeShader1DecisionSpec.kt:111` calls `it.clear()` then restores `PATH` and `HOME`. `committedToGit()` must do the same.

**Evidence:** `SpikeDecisionDocument.kt:24: pb.environment().clear()` — no PATH restoration. Contrast: `SpikeShader1DecisionSpec.kt:111: it["PATH"] = System.getenv("PATH") ?: "/usr/bin:/bin"`.

**Fix:** After `pb.environment().clear()`, add: `pb.environment()["PATH"] = System.getenv("PATH") ?: "/usr/bin:/bin"`.
