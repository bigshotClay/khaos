## Shadow 6 — Add TC-22: committedToGit() must read stdout on background thread (timeout not dead code)

**Source finding:** COR-01 [HIGH]

**Problem:** `readText()` is called synchronously before `waitFor(30, SECONDS)`. The timeout guard is unreachable while the process is alive. Git hang → infinite test stall.

**Shadow TC-22:**

```
TC-22: committedToGit() timeout is not dead code
- VERIFY: SpikeDecisionDocument.committedToGit() reads process stdout on a background thread
  (CompletableFuture.supplyAsync or equivalent) so that the waitFor() timeout fires
  while the read is in progress
- Acceptable patterns: CompletableFuture, coroutine, explicit Thread
- Unacceptable: readText() called on the same thread before waitFor()
```

**Test type:** Static code inspection
